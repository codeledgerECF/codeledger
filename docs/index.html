How it works

1

Scans your repo

Builds a dependency graph, git churn history, test mappings, and content index. All local.

2

Scores every file

9 weighted signals: keywords, centrality, churn, recency, test relevance, size, error infrastructure, and more.

3

Selects what matters

Picks the top files within a token budget. Deterministic — same task, same repo, same bundle.

4

Delivers to your agent

Writes a context bundle your agent reads immediately. No manual file discovery needed.

5

Isolates parallel sessions

Each Claude Code tab gets its own session — separate bundles, separate state, with overlap detection.

# Install and initialize
npm install -g @codeledger/cli
cd your-project
codeledger init

# One command — scan + score + bundle
codeledger activate --task "Fix null handling in user service"

# Your agent now has .codeledger/active-bundle.md
# with the right files, ranked by relevance.

# Running multiple agent tabs? Each gets its own session:
codeledger session-init
codeledger sessions
