Security

Security at CodeLedger

Security is foundational to what we build. CodeLedger handles the context that drives AI-assisted code changes — so trust is non-negotiable. Here's how we protect your data and what to do if you discover a vulnerability.

Our practices

🔒

Encryption in transit

All data between your machine and CodeLedger servers is encrypted using TLS 1.2 or higher. We enforce HTTPS across all endpoints and reject insecure connections.

🗄️

Encryption at rest

Stored data — including account information and usage telemetry — is encrypted at rest using AES-256. Database backups are encrypted with the same standard.

🏗️

No source code storage

CodeLedger processes your repository index locally on your machine. Your source code is never uploaded to or stored on our servers. Only aggregate metadata (file counts, scores, session summaries) is transmitted.

🔑

Access controls

Production systems are access-controlled with least-privilege principles. Engineers require MFA and role-based access to reach customer data. Access logs are retained and audited.

🛡️

License key security

License keys use an HMAC-SHA256 signed token format. Refresh tokens are verified with replay-attack protection (5-minute timestamp window). Keys are stored locally in your home directory, not in the cloud.

📦

Dependency management

We monitor dependencies for known vulnerabilities using automated scanning. Security patches are applied on a regular cadence, and critical fixes are prioritized for immediate release.

Responsible disclosure

We welcome security research on CodeLedger. If you discover a potential vulnerability, please report it to us privately before public disclosure so we can investigate and remediate it.

  • →Email your findings to security@codeledger.dev
  • →Include a clear description of the issue, reproduction steps, and potential impact
  • →We will acknowledge receipt within 48 hours and provide a resolution timeline
  • →We ask for 90 days to remediate before public disclosure

We do not currently offer a paid bug bounty program, but we genuinely appreciate responsible researchers and will credit you (with permission) in our changelog.

Questions

For general security questions or concerns, contact us at security@codeledger.dev. For support issues unrelated to security, use support@codeledger.dev.
